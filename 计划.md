# 基于多智能体的大学生个性化学习辅助系统 - 开发计划

| 版本 | 日期       | 作者   |
|------|------------|--------|
| v1.0 | 2026-05-03 | 项目组 |

---

## 总体时间规划

| 阶段 | 预计天数 | 说明 |
|------|----------|------|
| 第一阶段：项目结构搭建 | 2天 | 初始化前后端项目、数据库配置 |
| 第二阶段：后端业务服务开发 | 5天 | 用户、题库、技能树等核心模块 |
| 第三阶段：前端基础页面 | 5天 | 登录、画像、技能树、学习页 |
| 第四阶段：核心流程联调 | 6天 | 画像→技能树→资源→答疑→评估闭环 |
| 第五阶段：优化与完善 | 4天 | UI、测试、安全 |
| 第六阶段：部署 | 3天 | Docker、分布式部署 |
| **总计** | **25天** | |

---

## 第一阶段：项目结构搭建（第1-2天）

### 1.1 创建前后端项目骨架

| 任务 | 技术栈 | 说明 |
|------|--------|------|
| 前端项目初始化 | Vue3 + Vite + Element Plus + Axios + WebSocket | `npm create vite@latest webstare-frontend -- --template vue` |
| 后端项目初始化 | SpringBoot 3.x + Spring Security + MyBatis-Plus + MySQL 8.0 + Redis | 使用 Spring Initializr 或手动搭建 |
| 项目目录结构 | - | 前后端分离结构，根目录分别为 `frontend/` 和 `backend/` |

### 1.2 搭建数据库

| 任务 | 说明 |
|------|------|
| 安装 MySQL 8.0 & Redis | 本地或 Docker |
| 创建数据库 schema | `CREATE DATABASE webstare DEFAULT CHARACTER SET utf8mb4;` |
| 初始化核心表结构 | 执行 SQL 脚本（见下文） |

**核心表清单：**
- `user`（用户表）
- `user_profile`（用户画像表）
- `knowledge_tree_node`（技能树节点表）
- `question_bank`（题库表）
- `knowledge_tag`（知识点标签表）
- `study_record`（学习记录表）
- `exam_record`（测验记录表）
- `answer_detail`（答题明细表）

### 1.3 配置API网关与跨域

| 任务 | 说明 |
|------|------|
| 后端跨域配置 | SpringBoot `CorsConfiguration` 或 Nginx |
| AI 服务通信接口预留 | 定义统一的 RPC/HTTP 接口规范与 DTO |

## 第二阶段：后端业务服务开发（第3-7天）

### 2.1 用户模块（第3天）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/user/register` | POST | 用户注册 |
| `/api/user/login` | POST | 登录（返回 JWT token） |
| `/api/user/info` | GET | 获取用户信息 |
| `/api/user/profile` | GET/PUT | 获取/更新个人设置 |

### 2.2 题库模块（第4天）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/question/list` | GET | 题目列表（支持按标签/难度筛选） |
| `/api/question/{id}` | GET | 获取题目详情 |
| `/api/question/batch-import` | POST | 批量导入（Excel/CSV） |

### 2.3 技能树模块（第5-6天）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/skill-tree/generate` | GET | 获取/刷新用户个性化技能树 |
| `/api/skill-tree/node/{id}` | GET | 获取节点详情与资源 |
| `/api/skill-tree/node/{id}/learn` | POST | 标记节点「开始学习」 |
| `/api/skill-tree/node/{id}/test` | POST | 标记节点「直接测验」 |
| `/api/skill-tree/node/{id}/master` | POST | 标记节点「已掌握」 |

### 2.4 学习记录与测验模块（第7天）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/study/record` | POST | 记录学习行为 |
| `/api/exam/submit` | POST | 提交试卷 |
| `/api/exam/result/{id}` | GET | 获取测验结果 |
| `/api/exam/wrong-questions` | GET | 获取错题本 |

### 2.5 AI 服务调用层（贯穿本阶段）

封装与 AI agent 的通信接口（HTTP/RPC），定义如下 DTO：
- 画像对话请求/响应
- 资源生成请求/响应
- 智能答疑请求/响应
- 评估反馈请求/响应

## 第三阶段：前端基础页面（第8-12天）

### 8. 登录与欢迎页（第8天）
- 简洁欢迎界面
- AI 班主任头像与对话入口
- 登录/注册表单

### 9. 画像构建页（第9天）
- 流式对话组件（SSE）
- Markdown 画像卡片渲染
- 「没错，开始学习！」确认按钮

**技术选型：** `marked` + `highlight.js` 前端渲染 Markdown

### 10. 技能树可视化（第10天）
- 思维导图组件（推荐 `@antv/g6` 或 `vue3-mindmap`）
- 节点三态样式（红色-未学，黄色-学习中，绿色-已掌握）
- 节点悬浮菜单（【开始学习】【直接测验】【标记为已掌握】）

### 11. 学习资源页（第11天）
- 卡片墙布局
- Markdown 文档阅读器
- PPT/PDF 预览（避免直接生成视频）
- WebSocket 实时更新生成进度

### 12. 测验页与个人中心（第12天）
- 题目展示与作答
- 成绩反馈与错题重练
- 个人学习概览（ECharts 数据可视化）

## 第四阶段：核心流程联调（第13-18天）

### 13. 画像构建闭环（第13天）
```
用户对话 → 后端转发 → AI 画像分析智能体 → 画像入库 → 前端展示确认
```

### 14. 技能树生成闭环（第14天）
```
画像确认 → 调用 AI 路径规划智能体 → 返回树结构 JSON → 前端渲染
```

### 15. 资源生成闭环（第15-16天）
```
点击「开始学习」 → 调度中心拆解任务 → 文档/题库/多媒体智能体并行
→ WebSocket 实时推送进度 → 卡片流式展示
```

### 16. 智能答疑闭环（第17天）
```
用户提问 → AI 辅导智能体 → 答案（文字+图解） + 相似题目推送
→ 可选：薄弱点自动生成新节点
```

### 17. 评估反馈闭环（第18天）
```
测验结束 → AI 评估智能体 → 多维评估报告 → 更新画像 → 调整技能树状态
→ 解锁后续节点 → 推送下一阶段计划
```

## 第六阶段：部署（第23-25天）

### 23. Docker 镜像构建
- 前端：Nginx + 静态资源
- 后端：OpenJDK + SpringBoot JAR
- AI 服务：Python 镜像（由 AI 团队提供）

### 24. docker-compose 编排
一键启动：前端、业务服务、AI 服务、MySQL、Redis、向量库

### 25. 3节点分布式部署配置
| 节点 | 部署内容 |
|------|----------|
| 节点1 | 前端静态资源 + Nginx 反向代理 |
| 节点2 | SpringBoot 业务服务 + MySQL + Redis |
| 节点3 | AI 智能体服务 + 向量数据库 |

## 注意事项

1. **AI 层已完成**：重点是对接接口，定义好清晰的 DTO 协议
2. **视频不现实**：按文档建议，改为 PPT、文档、数字人对话
3. **随学随新**：画像更新要及时触发技能树和资源策略调整
4. **安全性**：生成内容必须经过事实性校验和敏感词过滤
