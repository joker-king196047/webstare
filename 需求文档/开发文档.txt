## 一、引言

### 1.1 项目背景

在高等教育数字化转型中，大学生学习呈现知识点碎片化、需求个性化、场景多样化等特征。传统学习平台存在资源匹配粗糙、辅导缺乏针对性、交互单一、学习状态难以持续跟踪等问题；同时，大模型应用普遍面临上下文遗忘、记忆混乱、复用性差、无法深度融合垂域业务等痛点，难以支撑稳定可靠的个性化学习服务。

### 1.2 项目目标

构建一套全自研、可信赖、分布式、业务深度融合的 AI 学习辅助系统，以多智能体协同的方式，结合动态学生画像、长期记忆引擎和个性化资源生成，实现从“登录即画像”到“学习闭环”的完整链路，真正落地“因材施教、精准辅导、持续成长”的智慧学习愿景。

### 1.3 适用范围

本系统面向高校计算机、人工智能、电子信息等相关专业学生，可被集成至校内教学平台或作为独立学习助手使用。

### 1.4 术语定义

- **多智能体**：系统中承担不同角色的 AI 代理（Agent），如班主任智能体、学霸智能体、出题智能体等，能够自主决策、相互通信、协同完成任务。
- **记忆引擎**：自研的长期记忆管理模块，采用向量数据库与关键信息结构化存储，为智能体提供用户历史上下文与知识快照。
- **技能树**：以思维导图形式呈现的结构化知识点图谱，既作为学习路径的导航界面，也是系统资源组织的骨架。
- **动态画像**：包含至少 6 个维度的学生数字模型，随着学习行为实时演化。

## 二、需求分析

### 2.1 功能性需求

#### 2.1.1 对话式学习画像自主构建
- 用户首次登录后，由 AI 班主任通过自然对话自动提取专业、年级、知识基础、薄弱点、学习目标、认知风格等信息。
- 构建不少于 6 个维度的动态画像（如知识基础、认知风格、易错点偏好、学习节奏、兴趣方向、短期/长期目标）。
- 画像支持随学随新，任何后续交互、测验、浏览行为均可触发更新。

#### 2.1.2 多智能体协同资源生成
- 系统需体现多智能体架构，至少包含班主任、规划师、文档生成器、题库生成器、多媒体生成器等角色。
- 结合学生画像与所选技能树节点，自动生成至少 5 种类型的个性化学习资源：
  - 个性化讲解文档（Markdown + 代码示例）
  - 知识点思维导图 / 技能树
  - 难度递升的练习题目
  - 拓展阅读材料
  - 多模态教学视频 / 动画 / PPT

#### 2.1.3 个性化学习路径规划与资源推送
- 依据技能树和用户画像，智能规划学习顺序，生成动态学习路径。
- 支持用户对技能树节点进行“开始学习”“直接测验”“标记已掌握”三种操作。
- 资源以卡片流形式推送到对应节点，涵盖文档、视频、题库、实操案例等。

#### 2.1.4 智能辅导与多模态答疑
- 在学习或测验过程中遇到疑问时，可随时唤起 AI 班主任或 AI 学霸进行解答。
- 解答形式包括文字、图解说明、短视频讲解。
- 支持从本地题库牵引相似题目，实现举一反三。
- 对反复出现的薄弱点，自动生成补充知识节点并插入技能树。

#### 2.1.5 学习效果评估与闭环优化
- 实时跟踪学习行为、测验成绩、提问记录。
- 多维度评估知识掌握度、能力提升度和学习投入度。
- 根据评估结果动态调整资源推送策略和后续学习计划，完成画像更新与路径重规划。

### 2.2 非功能性需求
1. **界面与交互**：美观简洁，支持流式输出、Markdown 渲染、多模态内容卡片化展示，符合现代 AI 产品交互规范。
2. **防幻觉与内容安全**：生成内容须经过事实性校验和敏感词过滤，确保无违规信息。
3. **响应性能**：智能体核心功能响应时间控制在合理范围（对话类 < 3s，资源生成提供进度追踪）。
4. **全自研可控**：AI 智能体层完全自研，不依赖 LangChain 等第三方框架，记忆引擎自主设计。
5. **分布式部署**：支持 2～3 台服务器的小型分布式部署，业务与 AI 服务隔离。

## 三、系统总体设计

### 3.1 设计原则
- **业务与智能分离**：SpringBoot 负责业务逻辑、事务管理；Python AI 层专注于多智能体调度与模型推理。
- **记忆内生**：所有智能体共用统一的记忆引擎，实现跨会话的上下文保持。
- **画像驱动**：所有生成和推荐均以动态画像为基础。
- **技能树导航**：前端以思维导图形式承载学习路径，后端以树形结构组织资源。

### 3.3 技术栈
- **前端**：Vue3, Vite, Element Plus, Axios, WebSocket
- **业务后端**：SpringBoot 2.7/3.x, Spring Security, MyBatis-Plus, MySQL 8.0
- **AI 层**：Python 3.10+, 自研多智能体调度引擎, 自研记忆引擎, 大模型 API (科大讯飞星火等), 向量数据库 (Milvus/Qdrant)
- **中间件**：Redis, Nginx, 可选 RabbitMQ/Kafka
- **部署**：Docker + Docker-Compose, 支持 2~3 节点轻量分布式

### 3.4 部署方案
- **节点 1**：前端静态资源 + Nginx 反向代理
- **节点 2**：SpringBoot 业务服务 + MySQL + Redis
- **节点 3**：AI 智能体服务 + 向量数据库 + 模型调用管理
- 服务间通过内网 HTTP/RPC 通信，AI 层可独立扩缩容。

## 四、功能模块详细设计

### 4.1 用户模块
- 注册/登录（支持学号 + 密码、手机验证码）
- 专业、年级、兴趣方向设置
- 个人学习概览（总学时、刷题量、正确率趋势、技能树点亮节点数）

### 4.2 题库与刷题模块
- 多学科题库，支持按知识点标签检索
- 三种刷题模式：顺序刷题、智能组卷（依据画像）、错题重做
- 自动判分，错题自动归入错题本并关联薄弱知识点
- 提供做题统计与正确率可视化

### 4.3 AI 多智能体核心模块（重点）

#### 4.3.1 智能体角色定义

| 智能体         | 职责                                                     | 触发条件                       |
|----------------|----------------------------------------------------------|--------------------------------|
| **调度中心/班主任** | 统筹任务、对话引导、进度推送                         | 用户登录/主动对话/阶段完成     |
| **画像分析智能体** | 提取特征、构建与更新画像，维护画像时效性              | 对话/学习行为发生变更时        |
| **路径规划智能体** | 生成技能树，规划学习顺序，响应节点操作                | 画像初始化/更新/节点操作时     |
| **题库生成智能体** | 根据画像与知识点智能组卷，支持难度递升与通过率预估   | 资源生成任务/测验请求时        |
| **文档生成智能体** | 编写个性化 Markdown 教程、实操案例                   | 资源生成任务                   |
| **多媒体生成智能体** | 生成教学视频脚本、PPT、动画讲解                     | 资源生成任务                   |
| **辅导答疑智能体** | 解答疑问，提供多种解释形式，牵引类似题目              | 用户提问/视频中点击提问时      |
| **评估反馈智能体** | 收集学习数据，多维评估，输出调整建议                  | 学习活动结束后/定时任务        |

#### 4.3.2 记忆引擎设计
自研记忆引擎由三部分组成：
1. **短期对话缓存**：Redis 存储当前会话的上下文窗口，确保交互连续性。
2. **长期画像记忆**：MySQL 存储结构化画像字段，向量库存储用户历史行为嵌入（用于检索相似经验）。
3. **知识快照**：对技能树上每个节点的学习记录生成摘要向量，支持“回忆”此前学习细节。

每次智能体调用记忆引擎，均可获得连贯的上下文与用户最新画像，彻底解决大模型“遗忘”问题。

#### 4.3.3 个性化技能树与路径规划
- **技能树生成**：路径规划智能体根据用户画像与课程知识图谱，渲染出可交互思维导图。
- **节点三态操作**：
  - **开始学习**：触发多智能体协同，生成该节点的全套资源。
  - **直接测验**：题库智能体依据画像生成针对性试卷，测验后根据结果更新节点状态。
  - **标记已掌握**：用户自主跳过，节点变绿，画像同步更新。
- **动态剪枝**：评估智能体发现节点掌握度足够高时，自动解锁后续依赖节点。

#### 4.3.4 多模态资源生成流程（以“Python 多态”为例）
1. 用户在技能树点击「开始学习」。
2. 调度中心将任务拆解并发至各生成智能体。
3. **文档智能体**生成《Python 多态进阶指南》，含避坑提示和动手环节。
4. **题库智能体**生成 3 组难度递升的试卷（简单 40%，中等 40%，困难 20%），并预估通过率基线。
5. **多媒体智能体**生成 5 分钟教学视频（AI 班主任+学霸出镜）及配套 PPT。
6. 所有资源以流式卡片推送到前端，学生可自由切换。

#### 4.3.5 智能辅导闭环
- **视频内交互**：观看视频时可暂停提问，答疑智能体结合视频画面圈点解答。
- **错题牵引**：答疑后自动推送本地题库中 2~3 道同知识点题目。
- **薄弱点生成**：若某知识点被反复提问，则自动新建技能树子节点，并生成补充材料。

#### 4.3.6 学习评估与反馈
评估智能体每日/每周分析数据，更新画像的 `知识掌握度`、`易错点` 等维度，并触发两类调整：
- **资源推送调整**：提高视频类或文本类资源优先级。
- **路径调整**：将掌握度低的节点重新标记为红色，调整学习顺序。

- Markdown 文档（含公式、代码块）

## 七、多智能体协同机制详述

### 7.1 协同框架
自研协同引擎基于 **事件驱动 + 任务队列** 模式。调度中心将复杂任务（如“为节点 A 生成全套资源”）拆解为多个并行子任务，分发给各执行智能体，并通过消息中间件或直接 RPC 通信。

### 7.2 关键协同流程时序（资源生成）
1. 调度中心接收请求 → 创建任务 ID，写入任务状态表。
2. 并行调用文档智能体、题库智能体、多媒体智能体的生成接口，携带画像压缩快照。
3. 各智能体在生成过程中，可随时向记忆引擎查询用户历史记录（如“之前 super() 总错”），调整生成策略。
4. 生成完毕，结果回写任务表，并通知前端通过 WebSocket 实时更新进度。
5. 调度中心收集全部结果后，推送一条整合消息给用户。

### 7.3 防幻觉与内容安全
- 所有生成结果经事实性校验模块（调用大模型二次核验或查询知识库对比）。
- 文本输出前通过敏感词过滤器。
- 题目生成强制要求附带答案解析，降低凭空捏造风险。

## 八、部署与运维

### 8.1 容器化部署
提供 docker-compose 编排文件，一键启动前端、业务服务、AI 服务、数据库、向量库。

### 8.2 监控告警
- 服务健康检查：`/actuator/health` 及 AI 服务心跳。
- 大模型调用成本监控：统计每日 Token 消耗与费用。
- 异常日志采集到 Elasticsearch，通过 Grafana 展示。

### 8.3 扩展性设计
- AI 服务无状态化，可水平扩展多个实例，通过 Nginx 负载均衡。
- 向量数据库选型支持分布式，满足未来大规模知识存储。

## 九、测试方案

### 9.1 功能测试
- 覆盖从登录、画像对话、技能树生成、资源生成、测验、答疑到评估报告的全流程。

- 模拟 500 并发用户同时生成资源，观察响应时间与任务队列积压情况。
- 确保 AI 对话流式输出首字延迟 < 2s。

### 9.3 安全性测试
- 接口鉴权测试，防 SQL 注入、XSS 攻击。
- 生成内容敏感词过滤效果验证。
